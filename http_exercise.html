HTTP and Security Exercise

1. What is the Hypertext Transfer Protocol (HTTP)

Hypertext transfer protocol (HTTP) is a communications protocol used for transferring documents like HTML. It is the main form of data exchange on the web and it works in a client server model which means requests are made by a user.

2. Why is HTTP called a stateless protocol?

HTTP is called a stateless protocol because the server does not hold any information between two different requests.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP

The first phase of establishing a connection is to initiate it in the transport layer which is usually transmission control protocol (TCP). Once this has been initiated and a connection has been made the client can then send the request to the server. The server will not send data unless there is a request.

4. Describe the parameters on a get and post request.

Get requests data from the server and post sends data to the server.

5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

Status code 200 means that the request was successful, and the information has been returned. Status code 404 means that the file has not been found and the server could not find the information in the request.

6. What are HTTP cookies? Give an example.

HTTP cookies are information about a website that is stored on the client’s computer. They can be used to save personalised information, manage logins, shopping carts and track the user’s behaviour.

7. Why are cookies a security issue?

Cookies can be viewed and are visible. They can also be used for cross site tracking. Cookies can build a profile of the user with all their information which can be a risk to their privacy.

8. Describe session hijacking and XSS

Session hijacking is where the user’s session is taken over by a hacker. This is done by the attacker having access to your session ID. It gives access to whatever the user was on and have the same privileges as the user. XSS stands for cross site scripting and it is a form of attack where code is injected into a website. This is done so that the attacker can compromise their interaction with the website and have full control and access of the data.

9. Describe Cross-site request forgery (CSRF)

CSRF is a form of attack where the user is tricked in to performing an action they did not mean to perform. An example of this could be the attacker could change the users email address or password. The attacker uses cookies to identify the user as there is no other way of identifying the user on the site.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

CORS requests is a system that allows a browser to load information from another source. Information like images, and style sheets can be requested cross site. When the browser requests to the server that is hosting the resources, it will send a pre-flight request. A pre-flight request ensures that server will allow the request.
